# Contributing

This repository is designed for learners first. Clear examples, kind explanations, and beginner-safe changes matter as much as code.

## Ways To Contribute

- Improve explanations in README files
- Fix typos, naming issues, and broken commands
- Add expected output screenshots or sample payloads
- Add small, focused examples for logs, metrics, traces, or Collector pipelines
- Improve OTCA study notes and practice questions

## Beginner-Friendly Workflow

1. Pick one day or one module.
2. Make a small, focused improvement.
3. Verify that the commands in the README still make sense.
4. Keep examples simple before making them advanced.
5. Explain the learning value of your change in plain language.

## Contribution Guidelines

- Prefer consistent folder names such as `day-02-instrumentation` and `context-propagation`.
- Keep README sections in this order when possible: overview, learning outcomes, prerequisites, steps, expected output, troubleshooting.
- Favor runnable examples over theory-only notes.
- If a config is reused across days, place it under `shared-resources`.
- If you replace older material, move the old version into `legacy` instead of silently discarding it.
